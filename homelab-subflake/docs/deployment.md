# Deployment

## Overview

The homelab-subflake uses [deploy-rs](https://github.com/serokell/deploy-rs) for
deploying NixOS configurations to hosts. Devshell commands (`deploy-$hostname`)
are auto-generated by the reused `devShellCmds` flake-parts module from the base
flake.

## How it works

### deploy-rs nodes

`deploy.nodes` is built from the same `hosts` attrset that drives
`nixosConfigurations`. For each host, `mkDeployRsNode` (in `flake-lib.nix`)
creates a node config that:

- Sets `hostname` to `$nodeName.$mgmtDomain` (from data-flake)
- Uses `root` as the SSH user
- Activates the NixOS configuration via deploy-rs

### Devshell commands

The `devShellCmds` flake-parts module (from `base.flake-modules.devShellCmds`)
reads `self.deploy.nodes` and generates per-host deployment commands in the
devshell. Configuration is in `perSystem`:

```nix
devshellCmds.deployment = {
  enable = true;
  useDeployRs = true;
  localDeployment = true;
  desktopNotifications = true;
};
```

Enter the devshell with `nix develop` to see available `deploy-*` commands.

## Adding a new host

1. Add the host to the `hosts` attrset in `flake.nix`:

```nix
hosts = {
  sodium = { role = "server"; extraModules = []; };
  newhost = { role = "server"; extraModules = []; };
};
```

2. Ensure the host exists in `data-flake.data.hosts.all` with a `system` attribute.
3. Create the host configuration at `hosts/newhost/configuration/`.
4. Run `nix develop` — a `deploy-newhost` command will be available automatically.
